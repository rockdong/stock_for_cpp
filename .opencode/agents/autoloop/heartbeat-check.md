# 心跳检查清单

## 每次心跳执行的检查

### 1. 任务队列检查

检查 `/Users/kirito/Documents/projects/agents/autoloop/tasks.json`

```
如果 pending 不为空:
  - 取出最高优先级任务
  - 检查 assignee Agent 是否 idle
  - 如果 idle，将任务分配给该 Agent
  - 更新 status.json，标记 Agent 为 working
```

### 2. Agent 状态检查

检查每个 Agent 的 `memory/YYYY-MM-DD.md`

```
遍历所有 Agent:
  - 检查是否有今日日志
  - 如果超过 24h 无更新，标记为 stale
  - 如果 Agent 标记任务完成，更新任务状态
```

### 3. 任务流转检查

检查 `inProgress` 中的任务

```
遍历 inProgress 任务:
  - 检查任务是否完成 (Agent 日志中标记)
  - 如果完成:
    - 移动到 completed
    - 检查 triggerNext，创建下游任务
  - 如果阻塞:
    - 移动到 blocked
    - 记录阻塞原因
```

### 4. 进度同步

更新 `status.json`

```
- 计算各状态任务数量
- 更新项目阶段 (根据完成的任务类型)
- 记录最后更新时间
```

---

## 检查频率建议

| 检查项 | 频率 | 说明 |
|--------|------|------|
| 任务队列 | 每次心跳 | 发现新任务立即处理 |
| Agent 状态 | 每 4 小时 | 防止 Agent 过久无响应 |
| 任务流转 | 每次心跳 | 保持流程顺畅 |
| 进度同步 | 每次心跳 | 实时更新项目状态 |

---

## 异常处理

### Agent 无响应
- 超过 24h 无更新，发送提醒
- 超过 48h 无更新，将任务重新分配

### 任务阻塞
- 记录阻塞原因
- 尝试解决阻塞 (如依赖任务卡住)
- 如无法解决，报告给用户

### 任务失败
- 记录失败原因
- 重新尝试或报告用户
- 根据优先级决定处理方式

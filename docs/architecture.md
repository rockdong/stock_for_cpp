## 模块说明

### 1. C++ 分析引擎 (核心)

**位置**: `cpp/`

**职责**:
- 技术指标计算 (MA, EMA, MACD, RSI, KDJ, BOLL, ATR)
- 策略分析执行
- 基本面评分筛选
- 定时任务调度
- 数据获取与存储

**技术栈**:
- CMake 构建系统
- TA-Lib 技术分析库
- spdlog 日志库
- nlohmann/json JSON 处理
- cpp-httplib HTTP 客户端

### 2. Node.js 飞书服务

**位置**: `nodejs/`

**职责**:
- 飞书机器人消息处理
- 分析结果推送
- 微信登录认证
- 飞书登录认证
- REST API 提供基础接口

**技术栈**:
- Express 4
- @larksuiteoapi/node-sdk
- better-sqlite3 / mysql2
- winston 日志

### 3. NestJS API 服务

**位置**: `backend/`

**职责**:
- RESTful API 提供
- 数据 CRUD 操作
- 用户管理
- Prisma ORM 数据访问

**技术栈**:
- NestJS 11
- Prisma 5
- TypeScript 5

### 4. Web 前端

**位置**: `web-frontend/`

**职责**:
- 分析结果可视化
- K 线图表展示
- 分析进度监控
- 用户登录

**技术栈**:
- React 18
- Vite 5
- TypeScript 5
- Tailwind CSS 3
- lightweight-charts
- Zustand

### 5. 微信小程序

**位置**: `miniprogram/`

**职责**:
- 移动端股票查看
- 分析信号展示
- 用户绑定

## 数据流

### 分析流程

```
1. C++ 定时调度触发 (20:00)
   ↓
2. 获取股票列表 (Tushare API)
   ↓
3. 基本面筛选 (Phase 1)
   ↓
4. 技术分析执行 (Phase 2)
   ↓
5. 分析结果存储 (MySQL)
   ↓
6. 飞书推送通知
```

### 用户请求流程

```
用户请求 → API 层 → 数据库查询 → 返回结果
         ↓
      分析引擎调用 (如需)
```

## 部署架构

### Docker 部署

```
Docker Compose
├── stock-cpp        # C++ 分析引擎
├── mysql            # MySQL 数据库
├── stock-bot        # Node.js 飞书服务
├── api-server       # NestJS API 服务
└── web-frontend     # Web 前端 (Nginx)
```

### 端口映射

| 服务 | 内部端口 | 外部端口 |
|------|----------|----------|
| mysql | 3306 | 3306 |
| stock-bot | 3000 | 3000 |
| api-server | 3001 | 3001 |
| web-frontend | 80 | 8880 |

## 技术选型理由

| 选择 | 理由 |
|------|------|
| C++ 分析引擎 | 高性能计算，TA-Lib 原生支持 |
| Node.js 飞书服务 | 飞书 SDK 官方支持，异步处理能力强 |
| NestJS API | 企业级框架，Prisma ORM 类型安全 |
| React + Vite | 快速开发体验，现代前端工具链 |
| MySQL 生产数据库 | 高并发支持，事务完整性 |
| SQLite 开发数据库 | 轻量级，无需配置 |

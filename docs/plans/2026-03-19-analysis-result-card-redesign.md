# 分析结果卡片重新设计

**日期**: 2026-03-19
**状态**: 已批准

## 背景

用户反馈「消息页面太丑了」，具体痛点：
- **信息展示问题**：文字排版混乱、数据堆砌没有层次
- **视觉效果问题**：颜色单调、缺少视觉分隔
- **交互体验问题**：按钮不统一、操作麻烦

优先改进 **分析结果** 页面。

## 设计方案

采用 **卡片分组模式**：按信号类型（买入/卖出/持有）分组，每组使用不同颜色主题，统一交互按钮。

## 详细设计

### 1. 卡片头部

```
┌─────────────────────────────────┐
│  📊 最新分析结果                │  ← 蓝色主题
├─────────────────────────────────┤
│  📅 2026-03-19 · 📈 共 15 条    │
│  ✅ 买入 5 · ❌ 卖出 3 · ⏸️ 持有 7 │
└─────────────────────────────────┘
```

**元素：**
- 标题：固定 "📊 最新分析结果"
- 日期：显示 trade_date
- 统计栏：买入/卖出/持有的数量统计

### 2. 信号分组

**买入信号组（绿色主题）：**
```
┌─────────────────────────────────┐
│  🟢 买入信号 (5只)              │  ← 绿色 header
├─────────────────────────────────┤
│  ▫️ **平安银行** `000001.SZ`    │
│     策略: 均线突破 · 日线 ☀️     │
│     [📊 日线] [📊 周线] [📊 月线]│
│  ─────────────────────────────  │
│  ▫️ **招商银行** `600036.SH`    │
│     ...                         │
└─────────────────────────────────┘
```

**卖出信号组（红色主题）：**
```
┌─────────────────────────────────┐
│  🔴 卖出信号 (3只)              │  ← 红色 header
├─────────────────────────────────┤
│  ...                            │
└─────────────────────────────────┘
```

**持有信号组（灰色主题）：**
```
┌─────────────────────────────────┐
│  ⚪ 持有 (7只)                  │  ← 灰色 header
├─────────────────────────────────┤
│  ...                            │
└─────────────────────────────────┘
```

### 3. 单条结果格式

```
▫️ **股票名** `代码`
   策略: 策略名 · 周期 周期图标
   [📊 日线] [📊 周线] [📊 月线]
```

**字段映射：**
| 显示内容 | 数据字段 | 格式 |
|---------|---------|------|
| 股票名 | name | 加粗 **name** |
| 代码 | ts_code | 行内代码 `code` |
| 策略名 | strategy_name | 普通文本 |
| 周期 | freq | d→日线☀️, w→周线⭐, m→月线🌙 |

### 4. 按钮交互

**统一提供三个按钮：**
| 按钮 | action_id | value |
|-----|-----------|-------|
| 📊 日线 | chart_d | {action:"chart", ts_code, freq:"d"} |
| 📊 周线 | chart_w | {action:"chart", ts_code, freq:"w"} |
| 📊 月线 | chart_m | {action:"chart", ts_code, freq:"m"} |

**交互逻辑：**
- 点击按钮 → 后端收到回调 → 生成对应周期图表 → 回复图片消息

### 5. 空状态设计

```
┌─────────────────────────────────┐
│  📊 最新分析结果                │  ← 灰色主题
├─────────────────────────────────┤
│  🔍 暂无分析结果                │
│                                 │
│  💡 发送「分析进度」查看运行状态 │
└─────────────────────────────────┘
```

## 技术实现

### 修改文件

1. **nodejs/src/formatters/analysisFormatter.js** - 重写 formatLatestAnalysis 函数

### 实现要点

1. 按 label 分组结果 (BUY/SELL/HOLD)
2. 每组使用独立卡片，不同 header template 颜色
3. 统一三个图表按钮，不依赖 opt 字段
4. 头部显示日期和统计信息

### 飞书卡片颜色映射

| 信号类型 | label 值 | header.template |
|---------|---------|-----------------|
| 买入 | BUY | green |
| 卖出 | SELL | red |
| 持有 | HOLD | grey |

# 股票分析机器人项目说明

## 项目概述

这是一个基于C++高性能数据处理引擎和Node.js Web服务的智能股票分析系统。系统通过Tushare API获取金融数据，进行技术分析和量化策略回测，并通过飞书机器人提供智能投资建议。

## 技术架构

```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│   Tushare API   │───▶│   C++ 数据引擎   │───▶│   SQLite 数据库  │
│  (金融数据源)    │    │  (高性能处理)     │    │  (股票数据存储)  │
└─────────────────┘    └──────────────────┘    └─────────────────┘
                                                        │
                                                        ▼
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│   飞书机器人     │◀───│  Node.js Web服务 │◀───│   REST API      │
│  (消息通知)      │    │  (业务逻辑)       │    │  (数据查询接口)  │
└─────────────────┘    └──────────────────┘    └─────────────────┘
```

## 核心模块

### 1. 数据获取模块 (`cpp/data/`)
- **DataSource**: 统一数据源接口，支持多种数据提供商
- **TushareClient**: Tushare API客户端，实现智能限流和重试机制
- **DataValidator**: 数据质量校验，确保数据完整性

### 2. 技术分析模块 (`cpp/analysis/`)
- **TechnicalIndicators**: 技术指标计算库(MACD、RSI、KDJ等)
- **StrategyEngine**: 策略引擎，支持自定义量化策略
- **Backtester**: 回测系统，验证策略有效性

### 3. 网络通信模块 (`cpp/network/`)
- **HttpClient**: 高性能HTTP客户端，支持连接池和超时控制
- **RateLimiter**: API调用频率限制器，防止触发限流
- **CircuitBreaker**: 熔断器模式，提高系统稳定性

### 4. Web服务模块 (`nodejs/src/`)
- **路由层**: RESTful API接口定义
- **服务层**: 业务逻辑处理和数据转换
- **中间件**: 认证、日志、错误处理等横切关注点

### 5. 通知系统 (`nodejs/src/reply.js`)
- **飞书集成**: 富文本消息发送和卡片消息
- **模板引擎**: 动态生成投资建议和分析报告
- **消息队列**: 异步消息处理，提高响应速度

## 开发规范

### 代码风格
- **C++**: 遵循Google C++ Style Guide，使用clang-format格式化
- **Node.js**: 使用Standard.js风格，配合ESLint静态检查
- **命名规范**: 使用驼峰命名法，常量使用全大写下划线分隔

### 提交规范
```

[optional body]

[optional footer]
```

示例：

### 测试要求
- **单元测试**: 核心算法和业务逻辑必须有对应单元测试
- **集成测试**: API接口和数据库操作需要集成测试覆盖
- **性能测试**: 数据处理模块需要基准测试验证

## 部署架构

### Docker容器化
```yaml
services:
  stock-analysis:
    image: rockdong/stock_for_cpp:latest
    build:
      context: .
      dockerfile: Dockerfile
    environment:
      - NODE_ENV=production
      - PORT=3000
    volumes:
      - ./stock.db:/app/stock.db:ro
    ports:
      - "3000:3000"
```

1. **代码提交** → 触发GitHub Actions
2. **多平台构建** → Ubuntu(macOS)并行编译C++

4. **容器构建** → 多阶段Docker构建
5. **镜像推送** → 自动推送到GitHub Container Registry

## 监控与运维

### 关键指标
- **API调用频率**: 监控Tushare API使用量，避免超限
- **系统响应时间**: Web服务和数据分析的性能指标
- **数据库性能**: SQLite查询优化和索引使用情况
- **错误率**: 各模块异常处理和恢复能力

### 日志管理
- **结构化日志**: 使用JSON格式记录操作日志
- **分级记录**: DEBUG、INFO、WARN、ERROR四个级别
- **日志轮转**: 按日期和大小自动轮转，避免占用过多空间

## 安全考虑

### 数据安全
- **API密钥加密**: 使用AES-256加密存储敏感信息
- **访问控制**: 基于角色的权限管理体系
- **数据脱敏**: 敏感信息在日志中自动脱敏处理

### 系统安全
- **依赖更新**: 定期更新第三方依赖，修复安全漏洞
- **输入验证**: 严格验证API参数，防止注入攻击
- **容器安全**: 使用最小化基础镜像，减少攻击面

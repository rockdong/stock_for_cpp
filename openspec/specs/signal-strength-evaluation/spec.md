# 信号强度评估规格说明

## 功能性需求

### 信号强度分级
- **需求**: 系统必须为每个交易信号提供强度分级评估
  - **场景**: 强信号识别
    - **当** EMA17/EMA25 黄金交叉且满足以下所有条件
      - 最近3天 EMA17 和 EMA25 均呈上升趋势
      - 成交量放大超过5日均量
    - **那么** 系统返回信号强度为 STRONG
    - **并且** 置信度 score >= 70
  
  - **场景**: 中等信号识别
    - **当** EMA17/EMA25 黄金交叉但部分条件满足
      - 趋势条件满足但成交量无明显放大
    - **那么** 系统返回信号强度为 MEDIUM
    - **并且** 置信度 score 在 40-69 之间
  
  - **场景**: 弱信号识别
    - **当** EMA17/EMA25 黄金交叉但趋势不明确
      - 仅发生交叉但趋势条件不满足
    - **那么** 系统返回信号强度为 WEAK
    - **并且** 置信度 score < 40

### 置信度计算
- **需求**: 系统必须为每个信号提供0-100的置信度评分
  - **场景**: 置信度基础计算
    - **当** 策略生成买入信号
    - **那么** 系统计算置信度基于以下因素
      - 趋势强度权重 40%
      - 成交量确认权重 30%
      - 技术指标共振权重 30%
  
  - **场景**: 最小置信度过滤
    - **当** 用户设置最小置信度阈值（如 min_confidence=50）
    - **那么** 系统过滤掉置信度低于阈值的信号
    - **并且** 返回空结果而非低质量信号

### 风险提示
- **需求**: 系统必须为信号提供风险提示信息
  - **场景**: 趋势反转风险提示
    - **当** 检测到 EMA 趋势即将反转（斜率趋近于0）
    - **那么** 系统在 risk_warning 字段中提示"趋势可能反转，注意风险"
  
  - **场景**: 高波动风险提示
    - **当** 股票近期波动率超过历史均值2倍
    - **那么** 系统在 risk_warning 字段中提示"波动率异常，谨慎操作"

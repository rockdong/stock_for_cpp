# 定时任务调度器设计方案

**日期**: 2026-03-07
**状态**: 已批准

## 1. 需求概述

为股票分析系统添加定时任务功能：每天晚上 8 点执行批量分析，仅在交易日（周一到周五）运行。

## 2. 架构设计

### 2.1 程序模式

| 模式 | 说明 | 默认 |
|------|------|------|
| 定时模式 | 启动后常驻，每晚 8 点执行 | ✅ |
| 单次模式 | 启动后执行一次然后退出 | ❌ |

### 2.2 模块结构

```
scheduler/
├── Scheduler.h/cpp      # 定时调度器核心
└── TradeCalendar.h/cpp  # 交易日历判断
```

### 2.3 主程序流程

```
main()
  ├── 解析命令行参数
  ├── 初始化系统
  ├── if (单次模式)
  │     └── 执行一次分析 → 退出
  └── if (定时模式)
        └── 进入调度循环
              └── 每秒检查：
                    ├── 当前时间 == 20:00?
                    ├── 交易日(周一到周五)?
                    └── 当天未执行过?
                        └── 执行分析 → 标记已执行
```

## 3. 详细设计

### 3.1 Scheduler 调度器

**功能**:
- 管理定时检查循环
- 记录每日执行状态（防止重复执行）
- 支持优雅退出（信号处理）

**核心逻辑**:
```cpp
class Scheduler {
public:
    void run();  // 主循环
    
private:
    bool shouldRunNow();       // 检查是否应该执行
    void markAsRun();         // 标记今日已执行
    bool hasRunToday();       // 检查今日是否已执行
    void executeAnalysis();   // 执行批量分析
    void sleepUntilNextCheck(); // 智能休眠
};
```

**调度策略**:
- 每秒检查一次当前时间
- 20:00 触发执行（可配置）
- 每天只执行一次
- 使用文件记录上次执行日期（持久化）

### 3.2 TradeCalendar 交易日历

**功能**:
- 判断当前是否为交易日

**判断逻辑**:
```cpp
bool isTradingDay(const std::string& date);  // date: YYYY-MM-DD

// 规则：
// 1. 周一到周五 → 交易日
// 2. 周六、周日 → 非交易日
// 3. 可扩展：添加节假日判断（未来需求）
```

### 3.3 命令行参数

| 参数 | 说明 |
|------|------|
| 无参数 | 默认定时模式，常驻运行 |
| `--once`, `-o` | 单次执行模式 |
| `--help`, `-h` | 显示帮助信息 |
| `--time`, `-t` | 指定执行时间（默认 20:00） |

### 3.4 信号处理

支持以下信号，实现优雅退出：
- `SIGINT` (Ctrl+C)
- `SIGTERM`

收到信号后：
1. 停止调度循环
2. 等待当前任务完成（如正在执行）
3. 清理资源
4. 正常退出

## 4. 数据流

### 4.1 每日执行状态持久化

在 `data/` 目录下创建 `.last_run` 文件：
- 内容：最后执行日期（YYYY-MM-DD）
- 每次启动时读取，判断今日是否已执行

### 4.2 日志输出

- 调度状态：`logs/scheduler.log`
- 分析日志：`logs/daily_analysis.log`

## 5. 配置文件

在 `.env` 中添加调度相关配置：

```env
# ========== 调度配置 ==========
# 执行时间（24小时制，格式：HH:MM）
SCHEDULER_EXECUTE_TIME=20:00
# 调度日志级别
SCHEDULER_LOG_LEVEL=INFO
```

## 6. 错误处理

| 场景 | 处理 |
|------|------|
| 批量分析失败 | 记录错误日志，跳过本次执行 |
| 网络异常 | 重试机制（同现有逻辑） |
| 数据库异常 | 记录错误，程序继续运行 |
| 磁盘空间不足 | 警告日志，程序继续运行 |

## 7. 兼容性

- 保留原有单次执行行为
- 通过命令行参数切换模式
- 现有功能不受影响
